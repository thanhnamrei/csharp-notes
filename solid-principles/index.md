# SOLID Principles Overview

Tóm tắt 5 nguyên tắc, link tới từng file chi tiết.

- [SRP](srp.md) — Single Responsibility Principle
- [OCP](ocp.md) — Open/Closed Principle
- [LSP](lsp.md) — Liskov Substitution Principle
- [ISP](isp.md) — Interface Segregation Principle
- [DIP](dip.md) — Dependency Inversion Principle

## Cheat Sheet Nhanh

- **SRP:** 1 class = 1 lý do thay đổi.
- **OCP:** Mở rộng bằng code mới, không sửa code cũ.
- **LSP:** Subclass thay thế base mà không phá behavior.
- **ISP:** Interface nhỏ, chuyên biệt; không ép client dùng method thừa.
- **DIP:** Phụ thuộc abstraction; high-level không phụ thuộc trực tiếp low-level.
